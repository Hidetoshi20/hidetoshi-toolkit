# 个人设备现状与规划

> 基于 `docs/thoughts/_Mac Mini Home Server vs. Air Workstation .md` 整理

## 核心设备清单

### 1. Mac mini (M4)
- **规格**: 256GB 存储 / 16GB 内存
- **当前状态**: 公司办公 (替代 Windows)
- **规划定位**: **家庭服务器 / HomeLab**
- **主要用途**: 
  - 7x24小时运行 Agentic AI (OpenClaw/Clawdbot)
  - 挂载外置硬盘运行虚拟机 (稳定 I/O)
  - 家庭算力中心

### 2. MacBook Air (M1)
- **规格**: 256GB 存储 / 16GB 内存
- **硬件状况**: 
  - 主板接口损坏，仅剩 **1个** 接口勉强正常。
  - **严重缺陷**: 大部分时间无法主动向外供电 (必须依赖带 PD 的显示器或有源拓展坞)。
- **当前状态**: 家用上网 / 偶尔外出
- **规划定位**: **流动办公 / 公司主力**
- **主要用途**: 
  - 日常代码编写 (UI/Frontend)
  - 会议与移动场景
- **痛点解决方案**:
  - 接口/供电不足: 公司 **必须** 配备一个支持高功率 PD 供电的有源拓展坞 (Active Dock)，否则 Air 将无法带动任何外设且面临断电风险。
  - 性能弱: 使用 VS Code Remote 连接公司 Windows/Linux 主机进行重构建。

### 3. 公司配机 (Windows)
- **状态**: 运行缓慢
- **用途**: 必须使用公司网络进行代码构建
- **规划**: 作为 Remote Build Server (构建服务器)，承担重负载编译任务

### 4. 外设
- **显示器 (家)**: 支持反向充电 (目前连接 Air)
- **外置硬盘**: 存储虚拟机与数据 (规划永久连接 Mac mini)

## 决策：设备布局调整 (Action Item)

- [ ] **迁移**: 将 Mac mini M4 从公司带回家。
- [ ] **重组**: 
  - **家**: Mac mini + 外置硬盘 + 显示器 = Always-on AI Server。
  - **公司**: MacBook Air + 拓展坞 = 移动办公终端。
- [ ] **配置**: 在公司配置 VS Code Remote 环境，打通 Air 到构建机器的链路。
