# PR Template · vibe coding

> Copy into GitHub/GitLab PR 描述，按需删除占位段落。

## Title
```
<type>: <feature/bug> — <short summary>
```

## Summary
- **Context**：为什么要改？关联 spec/issue 链接。
- **What**：本 PR 做了什么（按 step 或 commit 概括）。
- **How**：关键实现/架构决策、复用/新增的抽象。

## Test Plan
- [ ] Step 1 — <手测路径 + 预期>
- [ ] Step 2 — <loading/error/empty>
- [ ] Step 3 — <权限/角色/浏览器>

## Risk & Rollout
- **风险点**：<潜在副作用、依赖>
- **监控/报警**：<要关注的指标、日志>
- **灰度/回滚**：<feature flag、开关、回滚策略>

## Design Doc Sync
- **状态**：<已更新 design.md#章节 / 无需更新（理由）>
- **链接**：<design.md / architecture.md 摘要或 PR 中的变更部分>

## Links

- tasks.md / design doc：<链接>
- 相关 PR / 截图：<链接>
