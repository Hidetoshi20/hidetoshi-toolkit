# Prompt · 小改（文案 / 样式 / 轻量交互）

> 适用：文案更新、布局微调、按钮移动、轻量视觉调整等“一步完成”的改动。

## 0. 变更摘要
```text
请把以下描述整理为 3 行变更摘要，并额外注明设计文档影响：
1. 变更内容（含截图/组件名）
2. 是否影响 i18n / a11y / 交互（Tab 顺序、聚焦、快捷键）
3. 是否影响布局（响应式断点/主题）
4. 设计文档：<需更新的章节 / 无需更新 + 理由>
再列出 3 条从用户角度写的验收标准。
```

## 1. 风险评估 / 注意事项
```text
基于当前需求，请提醒我：
- 可能影响到的全局样式 / 主题 token / 公共组件
- 容易误伤的交互（例如 tooltip、点击区域）
- 需要同步调整的测试/截图/文案资源
输出为 checklist。
```

## 2. 上下文装载
```text
以下是相关代码：
- 组件/样式：<粘贴>
- 主题 token / 全局样式：<粘贴>
请基于此生成 patch plan，强调：
1. 复用现有类名 / token / variant
2. 只在必要文件里改动
3. 若需新增样式，说明命名与复用策略
```

## 3. 最小 patch
```text
请给出 <=30 行的统一 diff，覆盖以上变更。
要求：
- 不引入新的类名体系
- 若涉及文案，确保 i18n key 合规
- 简述变更说明与回归风险
```

## 4. 自测
```text
列出手测步骤：
1. 常规分辨率 + 暗色/亮色主题
2. 可访问性（键盘导航、屏幕阅读器提示）
3. 文案多语言或回退值
```

## 5. Commit / PR
```text
输出：
- 建议 commit message（chore/style/fix）
- PR 摘要 + Test Plan（引用自测）
- 风险提示（样式覆盖范围、i18n 回归）
- Design doc Sync（若涉及设计约束、组件指南，写明 design.md 更新情况）
```
